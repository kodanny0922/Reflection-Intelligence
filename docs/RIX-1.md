RIX-1｜Reflection Intelligence Index
（文件索引與架構說明）

一、文件目的

RIX-1 為 Reflection Intelligence（RI）體系之唯一索引文件，
用於說明 RI 相關文件的角色分工、適用情境與優先順序。

本文件不具規範效力，
不新增任何行為要求，
僅作為所有 RI 文件的閱讀入口與結構地圖。

二、RI 的一句話定義

Reflection Intelligence（RI）是一組用於延後判斷、界定邊界與控管風險的反思與治理機制，
依不同責任來源，分為中文反省模式（RI-C）與治理型反思模式（RI-G）。

三、文件總覽與職責分工

以下為目前 RI 體系中所有正式文件及其職責：

一、RIS-C1｜中文反省模式（定義文件）
職責：

定義 RI 在中文語境下的反省意義

規範反省模式的啟動、退出與核心原則

明確延後對錯、不進行價值裁決

適用情境：

使用者主動要求反省

希望檢查前提、假設、利益與後果

二、BRAM-C1｜權限與責任邊界（行為邊界文件）
職責：

規範 RI-C 啟動期間系統可／不可執行之行為

明確禁止建議、裁決與責任轉移

界定系統與使用者的責任邊界

適用情境：

RI-C 已啟動

需要判斷某一回應是否越權

三、RIS-C2｜中止與越界規範（退出條件文件）
職責：

規範 RI-C 必須中止或退出的情境

區分 Mandatory Exit 與 Soft Exit

規定中止時的系統行為

適用情境：

出現違法、裁決、建議或責任轉移請求

判斷是否必須立即停止反省模式

四、RIS-G1｜治理型反思模式（治理規範文件）
職責：

定義系統主動啟動之治理型反思機制

規範風險控管、行為限制與優先順序

明確不可協商、最高優先權原則

適用情境：

高風險、合規或安全相關請求

系統需主動限制或中止回應

五、RIW-C1｜中文反省模式白皮書（說明文件）
職責：

說明 RI-C 的設計背景與語義選擇

解釋中文「反省」與對錯判斷的差異

協助理解 RIS-C1 的設計動機

適用情境：

初次理解 RI-C

對外說明或內部教育

六、RIW-G1｜治理型反思白皮書（說明文件）
職責：

說明治理型反思存在的必要性

解釋不可協商與最高優先權的理由

補足 RIS-G1 的敘事理解層

適用情境：

對外說明 RIS-G1

避免誤解治理限制為價值介入

四、文件關係與優先順序

RI 文件之適用關係如下：

一、治理優先原則

當 RIS-G1 啟動時，具有最高優先權

任何 RI-C 模式即暫停適用

二、反省模式內部順序

RIS-C1 定義反省本體

BRAM-C1 界定行為邊界

RIS-C2 規範中止與退出

三、說明文件之性質

RIW-C1 與 RIW-G1 不具規範效力

僅用於理解與說明

不影響任何系統行為判定

五、如何使用本索引

建議閱讀順序如下：

初次接觸 RI：
RIX-1 → RIW-C1 → RIS-C1

使用中文反省模式：
RIS-C1 → BRAM-C1 → RIS-C2

理解治理限制與優先權：
RIW-G1 → RIS-G1

RIX-1 本身不需頻繁修改，
僅在新增或移除 RI 文件時更新。
